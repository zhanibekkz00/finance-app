# 🚀 Инструкция по запуску Backend

## Шаг 1: Настройка .env файла

1. В папке `backend` создай файл `.env` (если его нет)
2. Скопируй туда следующее и **замени на свои данные**:

```env
# Подключение к PostgreSQL
# Формат: postgresql://username:password@host:port/database
DATABASE_URL="postgresql://postgres:твой_пароль@localhost:5432/finance_app?schema=public"

# JWT Secret (любая случайная строка, минимум 32 символа)
JWT_SECRET="my-super-secret-jwt-key-change-this-in-production-12345"
JWT_EXPIRES_IN="7d"

# Порт сервера
PORT=3000
```

**Важно:** 
- Замени `postgres` на свой username PostgreSQL
- Замени `твой_пароль` на свой пароль PostgreSQL
- Если порт PostgreSQL не 5432, укажи правильный

---

## Шаг 2: Генерация Prisma Client

Открой PowerShell в папке `backend` и выполни:

```powershell
cd C:\Users\Jon\Desktop\flutter_project\backend
npx prisma generate
```

Это создаст Prisma Client для работы с базой данных.

---

## Шаг 3: Запуск Backend

В той же папке `backend` выполни:

```powershell
npm run start:dev
```

Ты увидишь:
```
🚀 Backend running on http://localhost:3000
```

**Backend запущен!** 🎉

---

## Шаг 4: Проверка работы

Открой новый PowerShell и проверь:

```powershell
# Проверка, что сервер работает
curl http://localhost:3000
```

Или открой в браузере: `http://localhost:3000`

---

## Шаг 5: Тест регистрации пользователя

```powershell
curl -X POST http://localhost:3000/auth/register `
  -H "Content-Type: application/json" `
  -d '{\"email\":\"test@example.com\",\"password\":\"password123\"}'
```

Должен вернуть JSON с `user` и `token`.

**Сохрани токен** - он понадобится для других запросов!

---

## Шаг 6: Тест получения категорий

```powershell
curl -X GET http://localhost:3000/categories `
  -H "Authorization: Bearer ТВОЙ_ТОКЕН_ЗДЕСЬ"
```

Должны вернуться категории: Food, Transport, Salary.

## 🐛 Проблемы?

### Ошибка: "Can't reach database server"
- Проверь, что PostgreSQL запущен
- Проверь `DATABASE_URL` в `.env`
- Убедись, что база `finance_app` существует

### Ошибка: "Prisma Client not found"
```powershell
npx prisma generate
```

### Порт 3000 занят
Измени `PORT=3001` в `.env` и перезапусти backend
