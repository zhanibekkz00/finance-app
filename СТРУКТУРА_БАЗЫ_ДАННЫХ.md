# Структура базы данных Finance App

## Обзор

Приложение использует SQLite базу данных для хранения транзакций и категорий. База данных создается автоматически при первом запуске приложения.

## Таблицы

### 1. Таблица `categories` (Категории)

Хранит информацию о категориях расходов и доходов.

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT PRIMARY KEY | Уникальный идентификатор категории |
| `name` | TEXT NOT NULL | Название категории (например, "Food", "Transport") |
| `colorValue` | INTEGER NOT NULL | Цвет категории в формате ARGB (например, 0xFFF44336) |
| `iconCode` | INTEGER NOT NULL | Код иконки Material Icons (например, 0xe25a) |
| `isDefault` | INTEGER NOT NULL | Флаг: является ли категория системной (1) или пользовательской (0) |

**Примеры данных:**
- Food (Еда) - красный цвет, иконка еды
- Transport (Транспорт) - синий цвет, иконка транспорта  
- Salary (Зарплата) - зеленый цвет, иконка денег

### 2. Таблица `transactions` (Транзакции)

Хранит все финансовые транзакции пользователя.

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT PRIMARY KEY | Уникальный идентификатор транзакции |
| `type` | TEXT NOT NULL | Тип транзакции: "income" (доход) или "expense" (расход) |
| `amount` | REAL NOT NULL | Сумма транзакции |
| `categoryId` | TEXT NOT NULL | ID категории (внешний ключ на таблицу categories) |
| `date` | TEXT NOT NULL | Дата транзакции в формате ISO 8601 |
| `note` | TEXT NOT NULL | Заметка/описание транзакции |
| `isRecurring` | INTEGER NOT NULL | Флаг повторяющейся транзакции (1 = да, 0 = нет) |
| `currency` | TEXT NOT NULL | Валюта (USD, EUR, RUB, KZT) |
| `recurrenceInterval` | TEXT NOT NULL | Интервал повторения: "daily", "weekly", "monthly", "none" |
| `nextOccurrence` | TEXT | Дата следующего повторения (может быть NULL) |
| `isPinned` | INTEGER NOT NULL | Флаг закрепленной транзакции (1 = закреплена, 0 = нет) |

**Связи:**
- `categoryId` → `categories.id` (FOREIGN KEY)

## Инициализация базы данных

При первом запуске приложения автоматически:

1. **Создаются таблицы** `categories` и `transactions`
2. **Заполняются начальные категории:**
   - Food (Еда)
   - Transport (Транспорт)
   - Salary (Зарплата)

## Расположение базы данных

- **Android/iOS:** В директории приложения
- **Windows/Linux:** В пользовательской директории данных
- **Web:** Используется in-memory база данных (данные не сохраняются между сессиями)

## Операции с базой данных

### Категории
- `getAllCategories()` - получить все категории
- Категории можно добавлять и удалять через UI

### Транзакции
- `getAllTransactions()` - получить все транзакции (сортировка по дате DESC)
- `insertTransaction()` - добавить новую транзакцию
- `updateTransaction()` - обновить существующую транзакцию
- `deleteTransaction()` - удалить транзакцию

## Модели данных

### CategoryModel
```dart
class CategoryModel {
  final String id;
  final String name;
  final int colorValue;  // ARGB цвет
  final int iconCode;    // Material Icons код
  final bool isDefault;
}
```

### TransactionModel
```dart
class TransactionModel {
  final String id;
  final TransactionType type;  // income или expense
  final double amount;
  final String categoryId;
  final DateTime date;
  final String note;
  final bool isRecurring;
  final String currency;
  final RecurrenceInterval recurrenceInterval;
  final DateTime? nextOccurrence;
  final bool isPinned;
}
```

## Особенности

1. **Внешние ключи:** Используется FOREIGN KEY для связи транзакций с категориями
2. **Типы данных:** 
   - Булевы значения хранятся как INTEGER (1 или 0)
   - Даты хранятся как TEXT в формате ISO 8601
   - Цвета хранятся как INTEGER в формате ARGB
3. **Индексы:** Автоматически создаются для PRIMARY KEY полей
